🚀 Quick Start

  • cd question-1/docker
  • docker compose up -d
  • Open http://localhost:3000
  • Wait ~2 min for Metabase init
  • Connect DB: host=postgres, port=5432

🔒 Security Mirror (AWS)

  • PostgreSQL has no host port binding
  • Only Metabase container can reach DB
  • Mirrors AWS: RDS has no public route
  • Credentials loaded from .env file
  • Volume persists data between restarts

🔒 Security Design

  • PostgreSQL never publicly accessible
  • DB subnets have zero internet route
  • Security Groups follow least-privilege
  • RDS accepts connections from ECS only
  • All storage encrypted at rest (AES-256)
  • TLS 1.3 enforced on ALB

🔑 Secret Management

  • Password auto-generated by Terraform (32 chars)
  • Stored in AWS Secrets Manager as JSON
  • ECS fetches via secrets: — never in env vars
  • IAM policy scoped to specific secret ARN
  • Accessed via VPC Endpoint (no NAT cost)
  • Recovery window: 7 days before permanent delete

⚡ High Availability

  • 2 AZs — ap-southeast-1a + 1b
  • RDS Multi-AZ with automatic failover
  • ECS auto-scaling (CPU > 70% / Memory > 80%)
  • ALB health check on /api/health
  • ECS deployment circuit breaker with auto-rollback
  • NAT Gateway per AZ for resilience

📊 Observability

  • CloudWatch Container Insights enabled
  • ECS container logs → /ecs/coraline-metabase-prod
  • RDS Enhanced Monitoring (60s interval)
  • RDS Performance Insights (7-day retention)
  • ALB access logs → S3 (90-day lifecycle)
  • Alarms: CPU, Memory, 5xx, Healthy Hosts, Storage

🌐 Network Topology

  • VPC CIDR: 10.0.0.0/16
  • Public: 10.0.1-2.0/24 (ALB, NAT GW)
  • Private App: 10.0.11-12.0/24 (ECS)
  • Private DB: 10.0.21-22.0/24 (RDS only)
  • VPC Endpoints: Secrets, ECR, CW, S3
  • DB subnet route table: no default route

🐳 Metabase Config

  • Image: metabase/metabase:v0.50.0
  • Launch type: ECS Fargate (serverless)
  • CPU: 1 vCPU | Memory: 2 GB
  • Health check: /api/health (startPeriod 120s)
  • Timezone: Asia/Bangkok (ICT UTC+7)
  • Auto-scaling: 1–4 tasks
